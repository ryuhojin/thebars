# 와이어프레임 화면 맵

## 공통 원칙

- 관리자와 고객 UI 모두 업무별 URL 하나만 사용한다.
- Compact·Medium·Wide는 같은 route component, query cache, form state와 mutation을 공유한다.
- 아래 화면 ID는 문서와 Codex 작업 보고에서 사용한다.
- `index.html#screen=화면키&viewport=compact|medium|wide`로 특정 상태를 직접 열 수 있다.

## 화면별 반응형 핵심

### 인증

- Wide: 제품 설명과 폼을 2열로 표시한다.
- Compact: 설명 영역을 생략하고 폼에 집중한다.
- 폼 필드와 제출 상태는 폭 전환으로 초기화하지 않는다.

### 관리자 셸

- Wide: 전체 사이드바와 테이블·분할 패널을 사용한다.
- Medium: 축소 내비게이션과 1~2열 레이아웃을 사용한다.
- Compact: drawer, 카드 목록, full-screen 편집, 하단 sticky action을 사용한다.

### 메뉴 관리

- 검색·필터·선택·일괄 변경 상태는 폭 전환 후 유지한다.
- drag-and-drop에는 버튼 또는 대상 선택 fallback이 있다.
- 메뉴 상세 탭과 입력값은 폭 전환으로 remount되지 않는다.

### 주문

- Wide: 주문 라인과 정산 패널을 나란히 표시한다.
- Compact: 주문 라인, 메뉴 추가 sheet, 하단 합계·정산 액션으로 전환한다.
- 모든 mutation은 version/expectedVersion과 idempotency 규칙을 지킨다.

### 고객 메뉴판

- Wide: 카테고리 레일과 2열 메뉴 카드를 사용한다.
- Compact: 가로 스크롤 카테고리와 1열 메뉴 카드를 사용한다.
- 제품 URL은 `/{encodedSlug}` 하나이며 관리자 API를 호출하지 않는다.
